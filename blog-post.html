作为技术支持或项目管理角色，你是不是也有过这样的困扰：每天处理了多少事务、支持了哪些项目，到了写周报的时候完全想不起来？ 我给自己写了一个小工具来解决这个问题——TicTracker，一个常驻 macOS 菜单栏的轻量计数器。

它能做什么

TicTracker 的核心理念很简单：用最低的操作成本，记录每天的工作量。
  • 菜单栏常驻：点一下就能看到今天的数据，不打断工作流
  • 全局快捷键：按 ⌃⇧+数字键 直接给对应项目 +1，手都不用离开键盘
  • 自定义项目：按你的实际工作分类，想怎么分就怎么分
  • 每日小记：除了计数，还能写几句今天做了什么
  • 本周趋势图：一眼看出这周的工作节奏，还有连续打卡天数
  • 一键生成周报：到了周五，点一下就把本周汇总复制到剪贴板
  • 日报提醒：每天定时提醒你记录，养成习惯

一些细节

在设置里可以：
  • 自定义项目名称，支持拖拽排序和重命名（历史数据自动迁移）
  • 修改快捷键的修饰键组合（⌃⇧ / ⌘⇧ / ⌥⇧ 等）
  • 自定义主界面的标题和小记标题
  • 设置日报提醒时间
  • 导出 / 导入 JSON 数据备份
支持日期切换，可以回看和编辑之前任意一天的数据，不用担心忘记记录。

技术实现

如果你也是开发者，可能会对技术栈感兴趣：
  • 纯 SwiftUI + Swift Package Manager 构建，没有用 Xcode 项目文件
  • Swift 6 严格并发模型，全局 @MainActor 隔离
  • 全局快捷键基于 Carbon API（RegisterEventHotKey）
  • 数据存储在 UserDefaults，轻量无依赖
  • 一个 build.sh 脚本完成编译、打包 .app、ad-hoc 签名
  • GitHub Actions CI/CD，打 tag 自动发布 Release
整个项目不到 1000 行代码，保持了足够的简洁。

为什么不用现有工具

市面上当然有很多时间追踪和计数工具，但大多数要么太重（Toggl、Clockify），要么需要联网和账号。我需要的只是一个：
开机就在，按一下就记，周五能出报告
的东西。所以就自己写了一个。

下载

项目开源在 GitHub：github.com/chenglun11/TicTracker 你可以直接从 Releases 页面下载最新版本，或者 clone 下来自己编译： 
git clone https://github.com/chenglun11/TicTracker.git
cd TicTracker
bash build.sh
open TicTracker.app
系统要求：macOS 15.0+ 如果觉得有用，欢迎 Star 或提 Issue。
